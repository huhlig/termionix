Telnet Server Implementation

This module provides a production-ready, async-first Telnet server implementation that addresses all critical issues identified in the V1 implementation:

  • No race conditions in connection management
  • Guaranteed resource cleanup with timeouts
  • Proper timeout handling for idle connections
  • Concurrent broadcast with backpressure
  • Lock-free metrics and monitoring
  • Clear separation of concerns

§Architecture

The implementation follows a layered architecture:

TelnetServer
    ↓
ConnectionManager
    ↓
ConnectionWorker → TelnetConnection

§Example

use termionix_server::{TelnetServer, ServerConfig, ServerHandler, ConnectionId, TelnetConnection};
use termionix_terminal::TerminalEvent;
use async_trait::async_trait;

struct MyHandler;

#[async_trait]
impl ServerHandler for MyHandler {
    async fn on_event(
        &self,
        id: ConnectionId,
        conn: &TelnetConnection,
        event: TerminalEvent,
    ) {
        // Handle events
    }
}

#[tokio::main]
async fn main() -> Result<(), Box<dyn std::error::Error>> {
    let config = ServerConfig::default();
    let server = TelnetServer::new(config).await?;
    server.start(std::sync::Arc::new(MyHandler)).await?;
    Ok(())
}
