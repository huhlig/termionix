Expand description

Terminal commands for output and control

This enum represents all commands that can be sent to a terminal, including text output, control sequences, and telnet commands.

Control(AnsiControlCode)

A C0 or C1 control character.

These are non-printable control codes in the ranges:

  • C0: 0x00-0x1F (excluding ESC at 0x1B)
  • DEL: 0x7F
  • C1: 0x80-0x9F

Common examples include NULL (0x00), Bell (0x07), Backspace (0x08), Tab (0x09), Line Feed (0x0A), and Carriage Return (0x0D).

§

AnsiEscape

A standalone ESC character (0x1B) that is not part of a recognized sequence.

This occurs when an ESC character is followed by a byte that doesn’t initiate a known ANSI escape sequence. The ESC was not consumed as part of a control sequence.

§

AnsiCSI(AnsiControlSequenceIntroducer)

Control Sequence Introducer - a general CSI escape sequence.

Format: ESC [ <params> <final_byte>

CSI sequences are used for cursor movement, screen manipulation, and other terminal control operations. The final byte (0x40-0x7E) determines the specific command. Common examples:

  • ESC[H - Cursor Home
  • ESC[2J - Clear Screen
  • ESC[10;20H - Move cursor to row 10, column 20

Note: SGR sequences (ending with ‘m’) are parsed separately and returned as the SGR variant instead.

§

AnsiSGR(AnsiSelectGraphicRendition)

Select Graphic Rendition - a specialized CSI sequence for text styling.

Format: ESC [ <params> m

SGR sequences control text appearance including colors, bold, italic, underline, and other visual attributes. This is a specialized form of CSI sequence that is parsed into a Style object for convenience.

Examples:

  • ESC[0m - Reset all attributes
  • ESC[1m - Bold
  • ESC[31m - Red foreground
  • ESC[1;31;42m - Bold red text on a green background
§

AnsiOSC(AnsiOperatingSystemCommand)

Operating System Command - a sequence for terminal-specific operations.

Format: ESC ] <params> ST or ESC ] <params> BEL

OSC sequences communicate with the terminal emulator to perform operations like setting the window title, changing color palettes, or other OS-level terminal features. The sequence is terminated by either ST (String Terminator, ESC ) or BEL (0x07).

The raw bytes (excluding the terminator) are returned for interpretation by the application.

§

AnsiDCS(AnsiDeviceControlString)

Device Control String - a sequence for device-specific control.

Format: ESC P <params> ST

DCS sequences are used to send device-specific control strings to the terminal. They are terminated by ST (ESC ). The contents are device- dependent and returned as raw bytes.

§

AnsiSOS(AnsiStartOfString)

Start of String - a legacy control sequence.

Format: ESC X <data> ST

SOS is a rarely used control function from ISO 6429. It marks the start of a control string that is terminated by ST (ESC ). The contents are returned as raw bytes.

§

AnsiST

String Terminator - marks the end of a string control sequence.

Format: ESC \

ST is used to terminate string-type control sequences (OSC, DCS, SOS, PM, APC). When encountered outside of a string sequence context, it’s returned as a standalone result with empty data.

§

AnsiPM(AnsiPrivacyMessage)

Privacy Message - a control sequence for private data.

Format: ESC ^ <data> ST

PM is a control function from ISO 6429 used to delimit privacy messages. The sequence is terminated by ST (ESC ) and the contents are returned as raw bytes.

§

AnsiAPC(AnsiApplicationProgramCommand)

Application Program Command - a control sequence for application-specific commands.

Format: ESC _ <data> ST

APC sequences allow applications to send custom commands through the terminal. The sequence is terminated by ST (ESC ), and the contents are returned as raw bytes for application-specific interpretation.

§

NoOperation

No Operation - Does nothing but sends an IAC command.

Format: IAC NOP (0xFF 0xF1)

This command is often used as a keep-alive or to test the connection.

§

DataMark

End of urgent Data Stream - Marks the end of urgent data.

Format: IAC DM (0xFF 0xF2)

Used to synchronize urgent and normal data streams in the connection.

§

Break

Break - Operator pressed the Break key or the Attention key.

Format: IAC BRK (0xFF 0xF3)

Sends an interrupt signal to the remote system, typically used to interrupt a running process on the server.

§

InterruptProcess

Interrupt Process - Request immediate interrupt of the current process.

Format: IAC IP (0xFF 0xF4)

Sends a stronger interrupt signal than Break, typically mapped to SIGINT in Unix-like systems.

§

AbortOutput

Cancel Output - Request that the remote system cancel output to the client.

Format: IAC AO (0xFF 0xF5)

Used when the client no longer wants to receive output from the remote process.

§

AreYouThere

Are You There - Request acknowledgment from the remote system.

Format: IAC AYT (0xFF 0xF6)

Used to test if the connection is still alive or if the remote host is responding.

§

EraseCharacter

Erase Character - Request that the operator erase the previous character.

Format: IAC EC (0xFF 0xF7)

Equivalent to sending a backspace or delete request to the remote system.

§

EraseLine

Erase Line - Request that the operator erase the previous line.

Format: IAC EL (0xFF 0xF8)

Requests the remote system to clear the current input line.

§

GoAhead

Go Ahead - End of input for half-duplex connections.

Format: IAC GA (0xFF 0xF9)

Used in half-duplex mode to indicate that the sender has finished transmitting and the receiver may now send data. Rarely used in modern systems.

§

EndOfRecord

End of Record - Marks the end of a prompt.

Format: IAC EOR (0xFF 0xEF)

Used by MUD servers to mark the end of a prompt. A prompt is considered any line that does not end with \r\n. This allows clients to distinguish between regular output and prompts that require user input.
