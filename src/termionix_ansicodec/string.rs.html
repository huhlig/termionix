28/// A mix of ASCII text, Unicode text, ANSI escape sequences/control codes, and Telnet Commands.
29/// Unlike [`StyledString`](crate::StyledString) which stores styling metadata separately,
30/// `SegmentedString` stores the raw segments themselves, making it ideal for:
31///
32/// - Parsing and preserving the exact structure of ANSI-formatted strings
33/// - Building terminal output incrementally
34/// - Precise control over ANSI sequence placement
35/// - Converting between different terminal representations
36///
37/// # Structure
38///
39/// Internally, `SegmentedString` is a wrapper around `Vec<Segment>`, where each [`Segment`]
40/// represents a distinct piece of content with its specific type (ASCII text, Unicode text,
41/// control codes, CSI sequences, etc.). This design allows:
42///
43/// - Efficient appending of content without parsing
44/// - Preservation of original ANSI sequence structure
45/// - Direct manipulation of individual segments
46/// - Conversion to other formats while maintaining semantic meaning
47///
48/// # Segment Types
49///
50/// A `SegmentedString` can contain the following types of segments:
51///
52/// - **ASCII**: ASCII text (0x20-0x7E, excluding ESC and control codes)
53/// - **Unicode**: Multi-byte Unicode text
54/// - **Control**: C0 or C1 control characters (e.g., newline, tab)
55/// - **Escape**: Standalone ESC character
56/// - **CSI**: Control Sequence Introducer commands (cursor movement, erasing, etc.)
57/// - **SGR**: Select Graphic Rendition (text styling like colors, bold, underline)
58/// - **OSC**: Operating System Commands (terminal title, etc.)
59/// - **DCS, SOS, ST, PM, APC**: Other ANSI escape sequence types
60///
61/// # Examples
62///
63/// Creating an empty segmented string:
64///
65/// ```rust
66/// use termionix_ansicodec::SegmentedString;
67///
68/// let mut segmented = SegmentedString::empty();
69/// assert!(segmented.is_empty());
70/// ```
71///
72/// Building a string with mixed content:
73///
74/// ```rust
75/// use termionix_ansicodec::{SegmentedString, Style, Color, ControlCode, Intensity};
76///
77/// let mut segmented = SegmentedString::empty();
78///
79/// // Add styled text
80/// segmented.push_style(Style {
81///     foreground: Some(Color::Red),
82///     intensity: Some(Intensity::Bold),
83///     ..Default::default()
84/// });
85/// segmented.push_str("Error: ");
86///
87/// // Add plain text
88/// segmented.push_str("File not found");
89///
90/// // Add a control character
91/// segmented.push_ansi_control(ControlCode::LF);
92/// ```
93///
94/// Building terminal output character by character:
95///
96/// ```rust
97/// use termionix_ansicodec::SegmentedString;
98///
99/// let mut segmented = SegmentedString::empty();
100/// for ch in "Hello".chars() {
101///     segmented.push_char(ch);
102/// }
103/// ```
104///
105/// # Segment Merging
106///
107/// `SegmentedString` intelligently merges consecutive compatible segments:
108///
109/// - Consecutive ASCII characters are merged into a single ASCII segment
110/// - Consecutive Unicode characters are merged into a single Unicode segment
111/// - ASCII and Unicode segments are merged when adjacent (promoted to Unicode)
112/// - Control codes, escape sequences, and style changes create new segments
113///
114/// This optimization reduces memory usage while preserving semantic meaning.
115///
116/// # Length Calculation
117///
118/// The [`len()`](SegmentedString::len) method calculates the display length based on
119/// the provided [`AnsiConfig`]. This takes into account which segments contribute to
120/// visible output versus terminal control. Most ANSI escape sequences do not contribute
121/// to display length.
122///
123/// # Comparison with Other Types
124///
125/// - [`SpannedString`](crate::SpannedString): For parsing ANSI strings and extracting
126///   byte ranges of each segment type. Returns immutable parse results with ranges.
127/// - [`StyledString`](crate::StyledString): For building styled text with automatic
128///   ANSI code generation. Stores text and styling separately.
129/// - `SegmentedString`: For building terminal output with explicit control over
130///   segments and ANSI sequences. Stores raw segments.
131///
132/// # Performance Considerations
133///
134/// - Adding characters/strings: O(1) amortized (may merge with last segment)
135/// - Counting segments: O(1)
136/// - Calculating length: O(n) where n is the number of segments
137/// - Memory: One allocation per segment (merged when possible)
138///
139/// # See Also
140///
141/// - [`Segment`] - The individual segment enum
142/// - [`SpannedString`](crate::SpannedString) - For parsing ANSI strings
143/// - [`StyledString`](crate::StyledString) - For building styled text
144#[derive(Clone, Debug, PartialEq, Eq)]

205    /// Appends a single character to the segmented string.
206    ///
207    /// This method intelligently handles both ASCII and Unicode characters, merging
208    /// them with the last segment when possible. Characters are classified as:
209    ///
210    /// - **ASCII**: Characters in range 0x20-0x7E (excluding ESC and control codes)
211    /// - **Unicode**: All other printable characters
212    ///
213    /// If the last segment is compatible (same type or can be promoted), the character
214    /// is appended to it. Otherwise, a new segment is created.
215    ///
216    /// # Segment Merging Rules
217    ///
218    /// - ASCII + ASCII → merged ASCII segment
219    /// - ASCII segment + Unicode character → promoted to Unicode segment
220    /// - Unicode + any character → merged Unicode segment
221    /// - After control/style → new text segment created
222    ///
223    /// # Arguments
224    ///
225    /// * `ch` - The character to append
226    ///
227    /// # Examples
228    ///
229    /// Basic usage:
230    ///
231    /// ```rust
232    /// use termionix_ansicodec::SegmentedString;
233    ///
234    /// let mut segmented = SegmentedString::empty();
235    /// segmented.push_char('H');
236    /// segmented.push_char('i');
237    /// ```
238    ///
239    /// Unicode characters:
240    ///
241    /// ```rust
242    /// use termionix_ansicodec::SegmentedString;
243    ///
244    /// let mut segmented = SegmentedString::empty();
245    /// segmented.push_char('世');
246    /// segmented.push_char('界');
247    /// ```
248    ///
249    /// Mixing ASCII and Unicode (promotes to Unicode):
250    ///
251    /// ```rust
252    /// use termionix_ansicodec::SegmentedString;
253    ///
254    /// let mut segmented = SegmentedString::empty();
255    /// segmented.push_char('A');   // ASCII segment
256    /// segmented.push_char('世');  // Promoted to Unicode segment
257    /// assert_eq!(segmented.segment_count(), 1); // Merged into one Unicode segment
258    /// ```
259    pub fn push_char(&mut self, ch: char) -> &mut Self {
260        // Check if character is ASCII (0x00-0x7F)
261        let is_ascii = ch.is_ascii() && ch != '\x1b' && !ch.is_control();
262
263        if let Some(last_segment) = self.0.last_mut() {
264            match last_segment {
265                Segment::ASCII(s) if is_ascii => {
266                    // Append ASCII character to ASCII segment
267                    s.push(ch);
268                }
269                Segment::ASCII(s) if !is_ascii => {
270                    // Convert ASCII segment to Unicode and append Unicode character
271                    let converted = std::mem::take(s);
272                    *last_segment = Segment::Unicode(converted);
273                    if let Segment::Unicode(unicode_str) = last_segment {
274                        unicode_str.push(ch);
275                    }
276                }
277                Segment::Unicode(s) => {
278                    // Append any character (ASCII or Unicode) to Unicode segment
279                    s.push(ch);
280                }
281                _ => {
282                    // Last segment is not a string segment, create a new one
283                    if is_ascii {
284                        self.0.push(Segment::ASCII(ch.to_string()));
285                    } else {
286                        self.0.push(Segment::Unicode(ch.to_string()));
287                    }
288                }
289            }
290        } else {
291            // No segments exist, create a new one
292            if is_ascii {
293                self.0.push(Segment::ASCII(ch.to_string()));
294            } else {
295                self.0.push(Segment::Unicode(ch.to_string()));
296            }
297        }
298        self
299    }

301    /// Appends a string slice to the segmented string.
302    ///
303    /// This method efficiently handles both ASCII and Unicode strings, merging them
304    /// with the last segment when possible. If the string is empty, this is a no-op.
305    ///
306    /// The entire string is classified as either ASCII or Unicode:
307    /// - **ASCII**: All characters are in range 0x20-0x7E
308    /// - **Unicode**: Contains any character outside the ASCII range
309    ///
310    /// # Segment Merging Rules
311    ///
312    /// - ASCII segment + ASCII string → merged ASCII segment
313    /// - ASCII segment + Unicode string → promoted to Unicode segment
314    /// - Unicode segment + any string → merged Unicode segment
315    /// - After control/style segment → new text segment created
316    ///
317    /// # Arguments
318    ///
319    /// * `str` - The string slice to append
320    ///
321    /// # Examples
322    ///
323    /// Basic usage:
324    ///
325    /// ```rust
326    /// use termionix_ansicodec::SegmentedString;
327    ///
328    /// let mut segmented = SegmentedString::empty();
329    /// segmented.push_str("Hello");
330    /// segmented.push_str(" World");
331    /// assert_eq!(segmented.segment_count(), 1); // Merged into one segment
332    /// ```
333    ///
334    /// Unicode strings:
335    ///
336    /// ```rust
337    /// use termionix_ansicodec::SegmentedString;
338    ///
339    /// let mut segmented = SegmentedString::empty();
340    /// segmented.push_str("こんにちは");
341    /// ```
342    ///
343    /// Mixed content:
344    ///
345    /// ```rust
346    /// use termionix_ansicodec::SegmentedString;
347    ///
348    /// let mut segmented = SegmentedString::empty();
349    /// segmented.push_str("Hello");  // ASCII segment
350    /// segmented.push_str("世界");   // Promoted to Unicode, merged
351    /// assert_eq!(segmented.segment_count(), 1);
352    /// ```
353    ///
354    /// Empty strings are ignored:
355    ///
356    /// ```rust
357    /// use termionix_ansicodec::SegmentedString;
358    ///
359    /// let mut segmented = SegmentedString::empty();
360    /// segmented.push_str("");
361    /// assert!(segmented.is_empty());
362    /// ```
363    pub fn push_str(&mut self, str: &str) -> &mut Self {
364        if str.is_empty() {
365            return self;
366        }
367
368        // Check if the entire string is ASCII
369        let is_ascii = str.is_ascii();
370
371        if let Some(last_segment) = self.0.last_mut() {
372            match last_segment {
373                Segment::ASCII(s) if is_ascii => {
374                    // Append ASCII string to ASCII segment
375                    s.push_str(str);
376                }
377                Segment::ASCII(s) if !is_ascii => {
378                    // Convert ASCII segment to Unicode and append Unicode string
379                    let converted = std::mem::take(s);
380                    *last_segment = Segment::Unicode(converted);
381                    if let Segment::Unicode(unicode_str) = last_segment {
382                        unicode_str.push_str(str);
383                    }
384                }
385                Segment::Unicode(s) => {
386                    // Append any string (ASCII or Unicode) to Unicode segment
387                    s.push_str(str);
388                }
389                _ => {
390                    // Last segment is not a string segment, create a new one
391                    if is_ascii {
392                        self.0.push(Segment::ASCII(str.to_string()));
393                    } else {
394                        self.0.push(Segment::Unicode(str.to_string()));
395                    }
396                }
397            }
398        } else {
399            // No segments exist, create a new one
400            if is_ascii {
401                self.0.push(Segment::ASCII(str.to_string()));
402            } else {
403                self.0.push(Segment::Unicode(str.to_string()));
404            }
405        }
406        self
407    }

1212    /// Returns the plain text content without any ANSI escape sequences or control codes.

1243    /// ```

1248    /// use termionix_ansicodec::{SegmentedString, Style, Color, Intensity};
1249    ///
1250    /// let mut segmented = SegmentedString::empty();
1251    /// segmented.push_style(Style {
1252    ///     foreground: Some(Color::Red),
1253    ///     intensity: Some(Intensity::Bold),
1254    ///     ..Default::default()

1258    /// segmented.push_str(": File not found");
1259    ///

1261    /// ```
1262    ///
1263    /// Removing control codes:
1264    ///

1266    /// use termionix_ansicodec::{SegmentedString, ControlCode};
1267    ///
1268    /// let mut segmented = SegmentedString::empty();
1269    /// segmented.push_str("Line 1");
1270    /// segmented.push_ansi_control(ControlCode::LF);
1271    /// segmented.push_str("Line 2");
1272    ///
1273    /// // Control codes are removed, only text remains
1274    /// assert_eq!(segmented.stripped(), "Line 1Line 2");
1275    /// ```
1276    ///
1277    /// Mixed ASCII and Unicode:
1278    ///
1279    /// ```rust
1280    /// use termionix_ansicodec::SegmentedString;
1281    ///
1282    /// let mut segmented = SegmentedString::empty();
1283    /// segmented.push_str("Hello");
1284    /// segmented.push_str(" 世界");

1286    /// assert_eq!(segmented.stripped(), "Hello 世界");

1290    ///
1291    /// ```rust
1292    /// use termionix_ansicodec::{SegmentedString, Style, Color, ControlCode, CSICommand};

1306    /// ```
1307    ///
1308    /// # Performance
1309    ///
1310    /// This operation is O(n) where n is the number of segments. It allocates a new
1311    /// `String` and concatenates only the text segments, making it efficient for
1312    /// extracting plain text from styled terminal output.
1313    ///
1314    /// # Use Cases
1315    ///
1316    /// - **Logging**: Save plain text logs without ANSI codes
1317    /// - **Text processing**: Extract content for analysis or search
1318    /// - **Testing**: Compare expected text content without worrying about styling
1319    /// - **Display**: Show content in environments that don't support ANSI codes
1320    /// - **Length calculation**: Get accurate character count of visible text
1321    ///
1322    /// # Comparison with Other Methods
1323    ///
1324    /// - [`styled_len()`](SegmentedString::len): Calculates display length with config
1325    /// - [`write_str()`](SegmentedString::write_str): Outputs with ANSI codes based on color mode

1337                Segment::ASCII(s) | Segment::Unicode(s) => Some(s.as_str()),

1341    }

1503    /// Removes and returns the last character from the segmented string.
1504    ///
1505    /// This method removes the last character from the last text segment (ASCII or Unicode)
1506    /// in the segmented string. If removing the character leaves the segment empty, the
1507    /// entire segment is removed. Returns `None` if the segmented string is empty or if
1508    /// the last segment is not a text segment.
1509    ///
1510    /// # Non-Text Segments
1511    ///
1512    /// If the last segment is not a text segment (Control, Escape, CSI, SGR, OSC, etc.),
1513    /// the entire segment is removed and returned as `None`. This ensures that control
1514    /// sequences and styling information are treated as atomic units.
1515    ///
1516    /// # Returns
1517    ///
1518    /// - `Some(char)` - The last character if the last segment contains text
1519    /// - `None` - If the string is empty or the last segment is not text
1520    ///
1521    /// # Examples

1527    ///
1528    /// let mut segmented = SegmentedString::empty();
1529    /// segmented.push_str("Hello");
1530    ///
1531    /// assert_eq!(segmented.pop(), Some('o'));
1532    /// assert_eq!(segmented.pop(), Some('l'));
1533    /// assert_eq!(segmented.stripped(), "Hel");
1534    /// ```
1535    ///
1536    /// Popping from an empty string:
1537    ///
1538    /// ```rust
1539    /// use termionix_ansicodec::SegmentedString;
1540    ///
1541    /// let mut segmented = SegmentedString::empty();
1542    /// assert_eq!(segmented.pop(), None);

1555    /// assert_eq!(segmented.pop(), Some('🦀'));

1559    /// Handling non-text segments:
1560    ///
1561    /// ```rust
1562    /// use termionix_ansicodec::{SegmentedString, ControlCode};
1563    ///
1564    /// let mut segmented = SegmentedString::empty();
1565    /// segmented.push_str("Hello");
1566    /// segmented.push_ansi_control(ControlCode::LF);
1567    ///
1568    /// // Control code is removed as a whole, returns None
1569    /// assert_eq!(segmented.pop(), None);
1570    /// assert_eq!(segmented.segment_count(), 1);
1571    ///
1572    /// // Now we can pop from the text segment
1573    /// assert_eq!(segmented.pop(), Some('o'));
1574    /// ```
1575    ///
1576    /// Mixed ASCII and Unicode segments:
1577    ///
1578    /// ```rust
1579    /// use termionix_ansicodec::SegmentedString;
1580    ///
1581    /// let mut segmented = SegmentedString::empty();
1582    /// segmented.push_str("Hello");
1583    /// segmented.push_str("世界");
1584    ///
1585    /// assert_eq!(segmented.pop(), Some('界'));
1586    /// assert_eq!(segmented.pop(), Some('世'));
1587    /// assert_eq!(segmented.pop(), Some('o'));
1588    /// ```
1589    ///
1590    /// Segment removal when empty:
1591    ///
1592    /// ```rust
1593    /// use termionix_ansicodec::SegmentedString;
1594    ///
1595    /// let mut segmented = SegmentedString::empty();
1596    /// segmented.push_str("AB");
1597    /// segmented.push_str("C");
1598    ///
1599    /// assert_eq!(segmented.segment_count(), 1); // Merged
1600    /// segmented.pop(); // Remove 'C'
1601    /// assert_eq!(segmented.segment_count(), 1); // Still has "AB"
1602    /// ```
1603    pub fn pop(&mut self) -> Option<char> {
1604        // Work backwards through segments to find the last text segment
1605        while let Some(last_segment) = self.0.last_mut() {
1606            match last_segment {
1607                Segment::ASCII(s) | Segment::Unicode(s) => {
1608                    // Try to pop a character from the text segment
1609                    if let Some(ch) = s.pop() {
1610                        // If the segment is now empty, remove it
1611                        if s.is_empty() {
1612                            self.0.pop();
1613                        }
1614                        return Some(ch);
1615                    } else {
1616                        // Empty a text segment, remove it and continue
1617                        self.0.pop();
1618                    }
1619                }
1620                _ => {
1621                    // Non-text segment (Control, Escape, CSI, SGR, etc.)
1622                    // Remove the entire segment and return None
1623                    self.0.pop();
1624                    return None;
1625                }
1626            }
1627        }
1628
1629        // No segments left
1630        None
1631    }

1633    /// Returns the display length of the segmented string based on the provided configuration.
1634    ///
1635    /// This calculates how many visible character positions the string occupies on screen,
1636    /// taking into account the ANSI configuration settings. Most ANSI escape sequences
1637    /// (control codes, CSI sequences, style changes) do not contribute to display length
1638    /// as they only affect formatting or cursor positioning.
1639    ///
1640    /// # Arguments
1641    ///
1642    /// * `config` - The ANSI configuration that determines how segments are interpreted
1643    ///
1644    /// # Returns
1645    ///
1646    /// The number of visible character positions the string occupies. This is the sum
1647    /// of the lengths of all text segments (ASCII and Unicode), excluding control
1648    /// sequences and formatting codes.
1649    ///
1650    /// # Examples
1651    ///
1652    /// ```rust
1653    /// use termionix_ansicodec::{SegmentedString, AnsiConfig};
1654    ///
1655    /// let mut segmented = SegmentedString::empty();
1656    /// segmented.push_str("Hello");
1657    ///
1658    /// let config = AnsiConfig::default();
1659    /// assert_eq!(segmented.len(Some(&config)).unwrap(), 5);
1660    /// ```
1661    ///
1662    /// With control codes (which don't contribute to display length):
1663    ///
1664    /// ```rust
1665    /// use termionix_ansicodec::{SegmentedString, AnsiConfig, ControlCode};
1666    ///
1667    /// let mut segmented = SegmentedString::empty();
1668    /// segmented.push_str("Hello");
1669    /// segmented.push_ansi_control(ControlCode::LF);
1670    /// segmented.push_str("World");
1671    ///
1672    /// let config = AnsiConfig::strip_all();
1673    /// assert_eq!(segmented.len(Some(&config)).unwrap(), 10); // Only counts "HelloWorld"
1674    /// ```
1675    pub fn len(&self, config: Option<&AnsiConfig>) -> AnsiCodecResult<usize> {
1676        let mut total_len = 0;
1677
1678        for segment in &self.0 {
1679            match segment {
1680                Segment::ASCII(s) | Segment::Unicode(s) => {
1681                    // For text segments, count the string length
1682                    total_len += s.len();
1683                }
1684                Segment::Control(_) => {
1685                    // Control codes contribute to length only if enabled in config
1686                    if let Some(config) = config
1687                        && config.strip_ctrl
1688                    {
1689                        // Strip CSI Sequence
1690                    } else {
1691                        total_len += 1; // Control codes are single bytes
1692                    }
1693                }
1694                Segment::Escape => {
1695                    total_len += 1; // Control codes are single bytes
1696                }
1697                Segment::CSI(_) => {
1698                    // CSI sequences contribute to length only if enabled in config
1699                    if let Some(config) = config
1700                        && config.strip_csi
1701                    {
1702                        // Strip CSI Sequence
1703                    } else {
1704                        total_len += 3; // Minimum: ESC [ char
1705                    }
1706                }
1707                Segment::SGR(sgr) => {
1708                    // SGR sequences contribute to length only if enabled in config
1709                    if let Some(config) = config {
1710                        if config.strip_sgr {
1711                            // Strip SGR Sequence
1712                        } else {
1713                            total_len += sgr.len(Some(config.color_mode));
1714                        }
1715                    } else {
1716                        total_len += sgr.len(None);
1717                    }
1718                }
1719                Segment::OSC(osc) => {
1720                    // OSC sequences contribute to length only if enabled in config
1721                    if let Some(config) = config
1722                        && config.strip_osc
1723                    {
1724                        // Strip OSC Sequence
1725                    } else {
1726                        total_len += osc.len();
1727                    }
1728                }
1729                Segment::DCS(dcs) => {
1730                    // DCS sequences contribute to length only if enabled in config
1731                    if let Some(config) = config
1732                        && config.strip_dcs
1733                    {
1734                        // Strip DCS Sequence
1735                    } else {
1736                        total_len += dcs.len();
1737                    }
1738                }
1739                Segment::SOS(sos) => {
1740                    // SOS/ST sequences contribute to length only if enabled in config
1741                    if let Some(config) = config
1742                        && config.strip_sos_st
1743                    {
1744                        // Strip SOS/ST Sequence
1745                    } else {
1746                        total_len += sos.len();
1747                    }
1748                }
1749                Segment::ST => {
1750                    // SOS/ST sequences contribute to length only if enabled in config
1751                    if let Some(config) = config
1752                        && config.strip_sos_st
1753                    {
1754                        // Strip SOS/ST Sequence
1755                    } else {
1756                        total_len += 2;
1757                    }
1758                }
1759                Segment::PM(pm) => {
1760                    // PM sequences contribute to length only if enabled in config
1761                    if let Some(config) = config
1762                        && config.strip_pm
1763                    {
1764                        // Strip PM Sequence
1765                    } else {
1766                        total_len += pm.len();
1767                    }
1768                }
1769                Segment::APC(apc) => {
1770                    // PM sequences contribute to length only if enabled in config
1771                    if let Some(config) = config
1772                        && config.strip_apc
1773                    {
1774                        // Strip APC Sequence
1775                    } else {
1776                        total_len += apc.len();
1777                    }
1778                }
1779                Segment::TelnetCommand(tc) => {
1780                    // Telnet Command sequences contribute to length only if enabled in config
1781                    if let Some(config) = config
1782                        && config.strip_telnet
1783                    {
1784                        // Strip TelnetCommand Sequence
1785                    } else {
1786                        total_len += tc.len();
1787                    }
1788                }
1789            }
1790        }
1791
1792        Ok(total_len)
1793    }

1954    /// # Use Cases
1955    ///
1956    /// - **Terminal output**: Display formatted text in terminals
1957    /// - **File generation**: Create ANSI-formatted text files
1958    /// - **Network protocols**: Send styled text over network connections
1959    /// - **Logging**: Output colored logs to terminal or file

1962    /// # See Also
1963    ///
1964    /// - [`ColorMode`] - Controls ANSI code generation
1965    /// - [`Style::write_style()`](AnsiSelectGraphicRendition::write) - Used internally for SGR segments
1966    /// - [`StyledString::write_str()`](crate::StyledString::write_str) - Similar method for `StyledString`
1967    pub fn write<W: std::io::Write>(
1968        &self,
1969        writer: &mut W,
1970        config: Option<&AnsiConfig>,
1971    ) -> std::io::Result<usize> {
1972        let mut total_len = 0;
1973        for segment in &self.0 {
1974            match segment {
1975                Segment::ASCII(text) | Segment::Unicode(text) => {
1976                    // Write plain text segments directly
1977                    writer.write_all(text.as_bytes())?;
1978                    total_len += text.len();
1979                }
1980                Segment::Control(control) => {
1981                    // Write control codes as their byte representation
1982                    writer.write_all(&[control.to_byte()])?;
1983                    total_len += 1;
1984                }
1985                Segment::Escape => {
1986                    // Write standalone ESC character
1987                    writer.write_all(b"\x1b")?;
1988                    total_len += 1;
1989                }
1990                Segment::CSI(csi) => {
1991                    if let Some(config) = config
1992                        && config.strip_csi
1993                    {
1994                        // Strip CSI Sequence
1995                    } else {
1996                        // Write CSI sequence
1997                        let len = csi.write(writer)?;
1998                        total_len += len;
1999                    }
2000                }
2001                Segment::SGR(sgr) => {
2002                    if let Some(config) = config {
2003                        if config.strip_sgr {
2004                            // Strip SGR Sequence
2005                        } else {
2006                            // Write SGR sequence
2007                            let len = sgr.write(writer, Some(config.color_mode))?;
2008                            total_len += len;
2009                        }
2010                    } else {
2011                        let len = sgr.write(writer, None)?;
2012                        total_len += len;

2097    /// Parses a string containing ANSI escape sequences into a `SegmentedString`.
2098    ///
2099    /// This method performs a complete analysis of the input string, identifying and
2100    /// converting all content into distinct owned segments. It handles ASCII text,
2101    /// Unicode characters, control codes, and all ANSI escape sequences (CSI, OSC, DCS, etc.),
2102    /// producing a `SegmentedString` ready for manipulation or rendering.
2103    ///
2104    /// # How It Works
2105    ///
2106    /// The parsing process occurs in two stages:
2107    ///
2108    /// 1. **Parse into spans**: Uses [`SpannedString::parse`] to analyze the input and
2109    ///    create lightweight byte range references to each segment type.
2110    ///
2111    /// 2. **Extract and own content**: Converts the spans into owned segments by extracting
2112    ///    the actual content from the source string using [`SpannedString::into_segmented_string`].
2113    ///
2114    /// This approach leverages existing, well-tested parsing infrastructure while producing
2115    /// a `SegmentedString` with fully owned data suitable for further manipulation.
2116    ///
2117    /// # Arguments
2118    ///
2119    /// * `str` - A string-like value that can be converted to `&str` via `AsRef<str>`.
2120    ///   This includes `&str`, `String`, `Cow<str>`, and other string types.
2121    ///
2122    /// # Returns
2123    ///
2124    /// A `SegmentedString` containing owned segments representing all content from the input:
2125    ///
2126    /// - **ASCII/Unicode segments**: Text content merged intelligently
2127    /// - **Control segments**: Individual control codes (newlines, tabs, etc.)
2128    /// - **CSI segments**: Parsed cursor/erase/scroll commands
2129    /// - **SGR segments**: Style and color information (extracted from CSI sequences)
2130    /// - **OSC/DCS/etc.**: Other escape sequence types with their data
2131    ///
2132    /// # Segment Types Produced
2133    ///
2134    /// The parser identifies and creates the following segment types:
2135    ///
2136    /// ## Text Segments
2137    ///
2138    /// - **ASCII**: Printable ASCII characters (0x20-0x7E)
2139    /// - **Unicode**: Multi-byte UTF-8 characters
2140    ///
2141    /// These are automatically merged when adjacent for efficiency. For example,
2142    /// "Hello World" becomes a single ASCII segment, and "Hello世界" becomes a single
2143    /// Unicode segment.
2144    ///
2145    /// ## Control Segments
2146    ///
2147    /// - **Control**: Terminal control characters like LF (`\n`), CR (`\r`), HT (`\t`), etc.
2148    /// - **Escape**: Standalone ESC character or unrecognized escape sequences
2149    ///
2150    /// ## ANSI Escape Sequences
2151    ///
2152    /// - **CSI**: Control Sequence Introducer (cursor movement, erasing, scrolling)
2153    /// - **SGR**: Select Graphic Rendition (colors, bold, underline, etc.)
2154    /// - **OSC**: Operating System Commands (window title, etc.)
2155    /// - **DCS**: Device Control String
2156    /// - **SOS**: Start of String
2157    /// - **ST**: String Terminator
2158    /// - **PM**: Privacy Message
2159    /// - **APC**: Application Program Command
2160    ///
2161    /// # Parsing Features
2162    ///
2163    /// ## Greedy Segment Merging
2164    ///
2165    /// The parser merges consecutive compatible segments to minimize memory usage:
2166    ///
2167    /// - Consecutive ASCII characters → Single ASCII segment
2168    /// - Consecutive Unicode characters → Single Unicode segment
2169    /// - ASCII + Unicode → Promoted to single Unicode segment
2170    /// - Consecutive identical control codes → Single Control segment
2171    ///
2172    /// ## SGR Extraction
2173    ///
2174    /// CSI sequences with the `m` command (SGR - Select Graphic Rendition) are
2175    /// automatically converted into `SGR` segments with parsed [`AnsiSelectGraphicRendition`] information,
2176    /// making it easy to work with colors and text formatting.
2177    ///
2178    /// ## UTF-8 Support
2179    ///
2180    /// Full Unicode support with proper UTF-8 character boundary detection. Multi-byte
2181    /// characters are handled correctly and merged into Unicode segments.
2182    ///
2183    /// ## Escape Sequence Termination
2184    ///
2185    /// String-type sequences (OSC, DCS, etc.) are properly terminated by:
2186    /// - ST (String Terminator): ESC \ or 0x9C
2187    /// - BEL (Bell): 0x07 (for OSC only)
2188    ///
2189    /// # Performance
2190    ///
2191    /// - **Time Complexity**: O(n) where n is the length of the input string
2192    /// - **Space Complexity**: O(m) where m is the number of segments produced
2193    /// - **Allocations**: One allocation per segment for text/data content
2194    /// - **Single Pass**: The string is scanned once during the span parsing phase
2195    ///
2196    /// The two-stage approach (parse spans, then extract content) is efficient because
2197    /// span parsing is fast (just tracking byte ranges), and content extraction happens
2198    /// only once per segment.
2199    ///
2200    /// # Examples
2201    ///
2202    /// ## Plain Text
2203    ///
2204    /// ```rust
2205    /// use termionix_ansicodec::SegmentedString;
2206    ///
2207    /// let segmented = SegmentedString::parse("Hello World");
2208    /// assert_eq!(segmented.segment_count(), 1);
2209    /// assert_eq!(segmented.stripped(), "Hello World");
2210    /// ```
2211    ///
2212    /// ## ANSI Colors
2213    ///
2214    /// ```rust
2215    /// use termionix_ansicodec::SegmentedString;
2216    ///
2217    /// let input = "\x1b[31mRed Text\x1b[0m";
2218    /// let segmented = SegmentedString::parse(input);
2219    ///
2220    /// // Produces: SGR(red) → "Red Text" → SGR(reset)
2221    /// assert!(segmented.segment_count() >= 3);

2224    ///
2225    /// ## Mixed Content with Control Codes
2226    ///
2227    /// ```rust
2228    /// use termionix_ansicodec::SegmentedString;
2229    ///
2230    /// let input = "Line 1\nLine 2\tTabbed";

2236    ///
2237    /// ## Unicode Content
2238    ///
2239    /// ```rust
2240    /// use termionix_ansicodec::SegmentedString;

2242    /// let input = "Hello 世界";

2253    /// use termionix_ansicodec::SegmentedString;
2254    ///
2255    /// let input = "\x1b[1;32mBold Green\x1b[0m Normal\n";

2257    ///
2258    /// // Produces multiple segments with styles, text, and control codes
2259    /// assert!(segmented.segment_count() > 3);
2260    /// ```
2261    ///
2262    /// ## Building from Parse
2263    ///
2264    /// ```rust
2265    /// use termionix_ansicodec::{SegmentedString, ControlCode};
2266    ///
2267    /// let mut segmented = SegmentedString::parse("\x1b[31mError:\x1b[0m ");

2269    /// segmented.push_ansi_control(ControlCode::LF);
2270    /// ```
2271    ///
2272    /// ## Generic String Types
2273    ///
2274    /// The method accepts any type implementing `AsRef<str>`:
2275    ///
2276    /// ```rust
2277    /// use termionix_ansicodec::SegmentedString;
2278    ///
2279    /// // From &str
2280    /// let s1 = SegmentedString::parse("text");

2282    /// // From String

2288    /// ```
2289    ///
2290    /// # Edge Cases
2291    ///
2292    /// - **Empty String**: Returns an empty `SegmentedString` with no segments
2293    /// - **Incomplete Sequences**: Treated as Escape segments containing the incomplete bytes
2294    /// - **Invalid UTF-8**: Individual bytes may create invalid segments (not validated)
2295    /// - **Malformed CSI**: Parsed as CSICommand::Unknown with available parameters
2296    /// - **Unterminated Strings**: OSC/DCS/etc. without ST extend to end of input
2297    ///
2298    /// # Use Cases
2299    ///
2300    /// This method is ideal for:
2301    ///
2302    /// - **Terminal Output**: Building terminal content from ANSI-formatted strings
2303    /// - **Format Conversion**: Converting ANSI strings to manipulable segments
2304    /// - **Content Filtering**: Extracting or modifying specific segment types
2305    /// - **Testing**: Verifying ANSI string composition and structure
2306    /// - **Rendering**: Preparing content for display with different color modes
2307    ///
2308    /// # Comparison with Other Parsing
2309    ///
2310    /// - **[`SpannedString::parse`]**: Returns lightweight byte ranges, no owned data
2311    /// - **`SegmentedString::parse`**: Returns owned segments ready for manipulation
2312    /// - **[`AnsiMapper`](crate::AnsiMapper)**: Byte-by-byte stateful parsing for streaming
2313    ///
2314    /// Use `SegmentedString::parse` when you need to:
2315    /// - Manipulate the parsed content (add, remove, or modify segments)
2316    /// - Build output incrementally after parsing
2317    /// - Convert between representations
2318    /// - Store parsed content for later use
2319    ///
2320    /// Use `SpannedString::parse` when you only need to:
2321    /// - Analyze structure without copying data
2322    /// - Extract specific portions by byte range
2323    /// - Perform read-only operations
2324    ///
2325    /// # See Also
2326    ///
2327    /// - [`SpannedString::parse`] - Lightweight parsing returning byte ranges
2328    /// - [`SegmentedString::push_str`] - Add text to an existing segmented string
2329    /// - [`SegmentedString::push_segment`] - Add individual segments
2330    /// - [`SegmentedString::stripped`] - Extract plain text without ANSI codes
2331    /// - [`StyledString`](crate::StyledString) - Alternative representation with style metadata

2434/// Represents a discrete segment of content within a [`SegmentedString`].
2435///
2436/// Each segment has a specific type that determines its purpose and how it should
2437/// be interpreted when rendering terminal output. Segments are the building blocks
2438/// of a `SegmentedString` and preserve the exact structure of ANSI-formatted strings.
2439///
2440/// # Variant Categories
2441///
2442/// ## Text Content
2443/// - [`ASCII`](Segment::ASCII) - Plain ASCII text (most common for English text)
2444/// - [`Unicode`](Segment::Unicode) - Multi-byte Unicode text (international characters, emoji)
2445///
2446/// ## Control Characters
2447/// - [`Control`](Segment::Control) - Terminal control characters (newline, tab, bell, etc.)
2448/// - [`Escape`](Segment::Escape) - Standalone ESC character
2449///
2450/// ## ANSI Escape Sequences
2451/// - [`CSI`](Segment::CSI) - Control Sequence Introducer (cursor, erasing, etc.)
2452/// - [`SGR`](Segment::SGR) - Select Graphic Rendition (colors, bold, underline, etc.)
2453///
2454/// ## Advanced Escape Sequences
2455/// - [`OSC`](Segment::OSC) - Operating System Command (window title, etc.)
2456/// - [`DCS`](Segment::DCS) - Device Control String
2457/// - [`SOS`](Segment::SOS) - Start of String
2458/// - [`ST`](Segment::ST) - String Terminator
2459/// - [`PM`](Segment::PM) - Privacy Message
2460/// - [`APC`](Segment::APC) - Application Program Command
2461///
2462/// # Examples
2463///
2464/// Text segments store their content directly:
2465///
2466/// ```rust
2467/// use termionix_ansicodec::Segment;
2468///
2469/// let ascii_segment = Segment::ASCII("Hello".to_string());
2470/// let unicode_segment = Segment::Unicode("世界".to_string());
2471/// ```
2472///
2473/// Control and styling segments carry semantic meaning:
2474///
2475/// ```rust
2476/// use termionix_ansicodec::{Segment, ControlCode, Style, Color};
2477///
2478/// let newline = Segment::Control(ControlCode::LF);
2479/// let red_text = Segment::SGR(Style {

2481///     ..Default::default()
2482/// });

2490    /// C0 or C1 Control character (0x00-0x1F, 0x7F-0x9F, excluding ESC)
2491    Control(AnsiControlCode),
2492    /// Single ESC character without a sequence
2493    Escape,
2494    /// CSI - Control Sequence Introducer (ESC [ ... final_byte)

2496    /// CSI SGR - Select Graphic Rendition (ESC [ ... final_byte])

2499    OSC(AnsiOperatingSystemCommand),
